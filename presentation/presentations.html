About this presentation

rcos.github.com/intro/presentations.html

Why we do project presentations

  • Learning who knows what/What's going on in RCOS
  • Speaking skills!
  • Show off!

Tech Talks vs Project Presentations

  • Tech talks can come from RCOS members or companies
  • Tech talks can be much longer (~15 minutes)
  • Talk to your mentor if you'd like to share a cool technology!

What should be in a project presentation

  • Project Name/Members/Mentor
  • What is your project? (short)
  • Progress Update/Significant Problems (short)
  • Demo!
  • Thanks!
  • Questions?
  • (synchronized clapping)

Can I do it this other way?

  • yea

Time

5 minutes

(or like 3 to 6)

Going over time

  • Coordinators will warn you
  • Tech talks/in depth presentations get more time
  • Talk to faculty/coordinators if you'd like more time
Section Recommended Time
Intro (Name/Members/Mentor) 30s
Project Description 30s
Progress 30s
Demo 1.5 mins
Thanks 15s
Questions 2 mins

Signing up

  • Sign up through your mentor
  • Mentors will stop you if you're not prepared

Quick tips

  • Be proud of your project
  • Great demos/walkthroughs speak for themselves
  • Start early! Last second signups make long RCOS meetings.
